# Guest Recovery Playbook

Recovery is a **ladder**, not a blank check. Classify failures and assign **fix / make / escalate** moves so comps stop being the only tool.

**Leading words:** recovery ladder · severity class · make-it-right not make-it-free · own the mic

## Workflow

1. **Severity class** — S1 annoyance · S2 failed expectation · S3 safety or dignity · S4 public / multi-guest. Map last-month examples into classes.
2. **Recovery ladder** — per class: acknowledge → fix path → **make-it-right not make-it-free** → follow-up. S3 safety: stop the dish, manager + log, never minimize.
3. **Comp authority matrix** — server / shift lead / GM caps + when to replate vs remove vs dessert vs return-visit card. Ban silent 100% comps without reason code.
4. **Own the mic (reviews)** — thank → own relevant fact → offline path → invite. Never fight strangers; never invent "we fired the cook."
5. **System fixes from patterns** — same S2 weekly → station or menu fix, not more free dessert.

## Controls

- Allergy / contamination: follow written protocol; if none, draft interim safe steps and require chef+owner ratification.
- No illegal discrimination in "problem guest" labeling.
- Third-party delivery refunds follow platform rules — do not invent policy.
- Recovery does not mean accepting guest harassment of staff.

## Deliverables

1. Severity taxonomy with house examples
2. Recovery ladder per class
3. Comp authority matrix
4. Review response bank (3–5)
5. Pattern → system fix list (top 3)
