# Staff Brain Backup

If it only lives in someone's head, the business is one resignation from amnesia. Capture **how work really happens** (not the handbook fantasy) into playbooks with **edge-case first** rules, then rate what is **agent-safe vs human-only**.

**Leading words:** how it really runs · edge-case first · definition of done · agent-safe vs human-only

## Workflow

1. **Map the real workflow** — trigger → steps → decisions → handoffs → tools → **definition of done**. Capture unofficial shortcuts people actually use; mark risky ones.
2. **Edge-case first catalog** — exceptions that consume judgment (difficult customer, rush order, partial payment, no-show, warranty gray zone). For each: signal → rule → escalate if → who owns.
3. **Write playbook pages** — one page per recurring job type: purpose, inputs, steps, checks, done criteria, examples, anti-patterns. Shop-floor simple language.
4. **Split agent-safe vs human-only** — Agent-safe: drafts, checklists, prep, lookups, summaries. Human-only: refunds over $, legal promises, HR, safety, gray-zone exceptions. Confidence: green / yellow / red.
5. **Backup package + drill** — playbook set, role FAQ, 5 quiz scenarios, 30-min fire drill while the expert is offline.

## Controls

- Interview for reality; do not invent process from industry stereotypes.
- Never encode discriminatory or illegal customer policies.
- Mark privacy fields (SSN, health, full payment PAN) as **do not paste into tools**.
- Prefer checklists over novels.

## Deliverables

1. Role workflow map
2. Edge-case catalog
3. Playbook pages by job type
4. Agent-safe / human-only matrix
5. Fire-drill quiz (5 scenarios)
