# Founder Time Reclaim

Most SMBs need the **owner unhooked from low-judgment work**, not more hustle. Build a **time truth table**, label E/D/A/K, pick one **reclaim wedge**, and design only the AI/automation slice with real ROI.

**Leading words:** time truth table · binary judgment · reclaim wedge · owner only #NO

## Workflow

1. **Build the time truth table** — rows = recurring tasks; columns: hours/wk | energy (H/M/L) | skill (owner-only?) | error cost | customer-facing? | currently who. Force **binary judgment** on owner-only (yes/no).
2. **Tag each row E / D / A / K** — Eliminate (no material value) · Delegate (human + checklist) · Automate (rules/AI, low judgment) · Keep (owner only — strategy, key relationships, rare exceptions).
3. **Pick one reclaim wedge (not ten)** — the single change that frees the most hours with least drama in 14 days (inbox triage, quote template + approval limit, appointment confirmation, bill-pay batching, FAQ deflection).
4. **Design the automation slice** — trigger → inputs → decision rule → human checkpoint → output → failure path. Prefer boring reliability over clever agents.
5. **Publish the owner operating system** — weekly: 3 owner-only blocks. Daily: max 2 exception reviews. Escalation SLA: staff try X before pinging owner.

## Controls

- **Owner only #NO** is sacred — protect few keep tasks; kill everything that steals them.
- No "buy 8 new tools." Prefer one workflow inside the stack they already pay for.
- If hours are guesses, run a 3-day measurement sprint before locking the wedge.
- Never automate refunds, contracts over agreed $, HR discipline, or privacy-sensitive acts without explicit human approval rules.

## Deliverables

1. Time truth table
2. E/D/A/K labels + hours recovered estimate
3. One 14-day reclaim wedge checklist
4. Automation slice (text diagram)
5. Owner weekly calendar skeleton
